# PRD — Согым Боты (Telegram)

## 1. Обзор продукта

**Название:** Согым Боты  
**Платформа:** Telegram Bot  
**Версия:** 1.0 MVP  
**Дата:** 2026-05-30  

### Проблема
Организация совместной покупки согыма (целой туши) — трудоёмкий процесс:
- Нужно вручную собирать участников через WhatsApp/звонки
- Нет прозрачности: кто взял долю, кто заплатил
- Организатор тратит много времени на координацию
- Часто возникают конфликты из-за неоплаченных долей

### Решение
Telegram-бот, который автоматизирует весь процесс — от создания закупки до подтверждения оплаты.

---

## 2. Целевая аудитория

| Сегмент | Описание |
|---|---|
| **Организатор** | Человек, который инициирует закупку согыма, договаривается с продавцом |
| **Участник** | Семья или человек, который хочет взять 1 долю (1/4 туши) |
| **География** | Казахстан. Активные пользователи Telegram |
| **Язык** | Казахский (основной), русский (вторичный) |

---

## 3. Пользовательские сценарии (User Stories)

### Организатор
- Как организатор, я хочу **создать согым** с названием, видом скота, ценой и датой, чтобы люди могли присоединиться
- Как организатор, я хочу **видеть список участников** и статус оплаты каждого
- Как организатор, я хочу **отметить оплату** участника вручную
- Как организатор, я хочу **закрыть согым** когда все доли разобраны

### Участник
- Как участник, я хочу **видеть список открытых согымов** с количеством свободных долей
- Как участник, я хочу **записаться на долю** в один клик
- Как участник, я хочу **выйти** из согыма если передумал
- Как участник, я хочу **видеть детали**: цену, дату, место забоя

## 4. Функциональные требования

### 4.1 Управление согымом

| ID | Функция | Приоритет |
|---|---|---|
| F-01 | Создание нового согыма (название, вид скота, цена/кг, дата, место) | 🔴 Must |
| F-02 | Выбор вида скота: Сиыр 🐄, Жылқы 🐎, Қой 🐑 | 🔴 Must |
| F-03 | Автоматический расчёт: кол-во долей, вес, стоимость 1 доли | 🔴 Must |
| F-04 | Закрытие/завершение согыма организатором | 🔴 Must |
| F-05 | Список активных согымов с прогресс-баром | 🔴 Must |
| F-06 | Детальный просмотр согыма: участники, остатки долей | 🔴 Must |

### 4.2 Участие

| ID | Функция | Приоритет |
|---|---|---|
| F-07 | Запись на 1 долю (кнопка "Қосылу") | 🔴 Must |
| F-08 | Отмена своей доли (кнопка "Шығу") | 🔴 Must |
| F-09 | Защита от двойной записи | 🔴 Must |
| F-10 | Запись на 2+ доли одним участником | 🟡 Should |

### 4.3 Оплата

| ID | Функция | Приоритет |
|---|---|---|
| F-11 | Организатор отмечает оплату участника | 🔴 Must |
| F-12 | Статус оплаты: ✅ оплачено / ⏳ ожидается | 🔴 Must |
| F-13 | Автоматический расчёт суммы к оплате на каждого | 🔴 Must |
| F-14 | Напоминание должникам (уведомление в боте) | 🟡 Should |
| F-15 | Интеграция с Kaspi QR | 🟢 Nice |

### 4.4 Уведомления

| ID | Функция | Приоритет |
|---|---|---|
| F-16 | Уведомление организатору когда участник записался | 🟡 Should |
| F-17 | Уведомление когда все доли разобраны | 🟡 Should |
| F-18 | Уведомление участникам об изменении даты/места | 🟡 Should |

## 5. Нефункциональные требования

| Параметр | Требование |
|---|---|
| **Язык интерфейса** | Казахский (основной), можно добавить переключение на русский |
| **База данных** | SQLite для MVP, PostgreSQL для продакшна |
| **Доступность** | 99% uptime (бот должен работать без остановок) |
| **Время ответа** | < 1 секунды на любую команду |
| **Безопасность** | Только организатор управляет оплатой и закрытием |

## 6. Виды скота и параметры по умолчанию

| Вид | Символ | Доли | Вес (кг) | Цена/кг (тг) | Стоимость 1 доли |
|---|---|---|---|---|---|
| Сиыр (Корова) | 🐄 | 4 | ~200 кг | 2 500 | ~125 000 тг |
| Жылқы (Лошадь) | 🐎 | 4 | ~220 кг | 3 000 | ~165 000 тг |
| Қой (Овца) | 🐑 | 2 | ~45 кг | 2 000 | ~45 000 тг |

> Организатор может изменить цену/кг вручную при создании согыма.

## 7. UX Flow

```
/start
  └─► Главное меню
        ├─► [Жаңа согым ашу]
        │     ├─► Ввод названия
        │     ├─► Выбор вида скота
        │     ├─► Ввод цены/кг
        │     ├─► Ввод даты
        │     ├─► Ввод места
        │     └─► ✅ Согым создан → карточка
        │
        ├─► [Белсенді согымдар]
        │     └─► Список → [Выбрать согым]
        │               ├─► Детали согыма
        │               ├─► [Қосылу] / [Шығу]
        │               └─► [Жабу] (только организатор)
        │
        └─► [Менің согымдарым]
              └─► Созданные + участвую
```

## 9. Структура данных

```sql

total_shares, event_date, location,
```

```sql
shares, paid (0/1), joined_at
```
